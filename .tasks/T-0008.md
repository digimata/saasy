## Objective

Replace the vendored AuthView sign-in/sign-up pages and auto-bootstrap setup with the custom auth UX from `dremnik/microprojects`. Same visual style: floating centered layout, animated transitions, email OTP flow, OAuth row, manual workspace creation.

## Reference files

Source repo: `dremnik/microprojects` (clone to `/tmp/microprojects/`)

| File | What to port |
|---|---|
| `src/app/sign-in/[[...sign-in]]/page.tsx` | Sign-in — email form, OTP verification, OAuth, animated transitions |
| `src/app/onboarding/page.tsx` | Create workspace — name + slug inputs, create button |
| `src/app/onboarding/layout.tsx` | Onboarding layout (minimal) |
| `src/app/onboarding/_actions.ts` | Server action for workspace creation |
| `src/components/ui/input-bubble.tsx` | Rounded input with subtle shadow |
| `src/components/ui/url-input.tsx` | Input with inline URL prefix |
| `src/components/ui/icons.tsx` | SVG icons (Google, arrow, chevron, etc.) |

## Sign-in page (`/sign-in`)

- No card wrapper — content floats on `bg-background`, centered, `pt-[24vh]`
- Logo SVG at top (replace Vercel triangle with saasy logo or placeholder)
- Heading: "Welcome to Saasy." (27px, medium weight)
- Email input using `InputBubble` (rounded-xl, subtle shadow, arrow button inside)
- Arrow button opacity changes based on email validity
- OAuth buttons row below (Google, GitHub — horizontal, icon-only, equal width)
- On email submit: BetterAuth email OTP flow (not Clerk)
- Animated transition (framer motion `AnimatePresence`) to OTP step

## OTP verification step (same page, animated swap)

- Logo + "Enter the code you received" heading
- "We've sent an email with a verification code to {email}" subtitle
- 6-digit `InputOTP` with auto-submit on complete
- "Verify your email" button
- Back button to return to email step
- Works for both sign-in and sign-up (BetterAuth determines which)

## Create workspace page (`/setup`)

- "Logged in as: {email}" + sign out link in top header bar
- Logo + "Create a workspace" heading (same centered layout)
- `InputBubble` for workspace name
- `UrlInput` for workspace slug with prefix (e.g. `saasy.app/`)
- Auto-generate slug from name
- "Create workspace" button with loading spinner
- On success: set active org, redirect to dashboard
- Replace current auto-bootstrap with this manual form

## What to remove

- Keep `AuthView` for forgot-password, email-verification, etc.

## Auth flow (Clerk → BetterAuth)

- Email submit: `authClient.emailOtp.sendVerificationOtp({ email, type: "sign-in" })`
- OTP verify: `authClient.emailOtp.verifyEmail({ email, otp })`
- If user doesn't exist on sign-in, auto-switch to sign-up
- OAuth: `authClient.signIn.social({ provider: "google" | "github" })`
- Create workspace: `authClient.organization.create()` + `authClient.organization.setActive()`
