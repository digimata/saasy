# T-0013 :: Design system — colors, typography, icons

## Objective

Establish a cohesive design system for saasy covering color tokens, type scale, and icon set.

## Scope

### 1. Color system
- Numbered scales (100–900) for each color family
- Core palette: gray + brand accent + semantic (success, warning, error, info)
- Semantic tokens that reference into scales (components use `--color-border`, not `--gray-400`)
- Light/dark mode mappings via CSS custom properties in `globals.css`
- Replace ad-hoc `mil-green` / `charcoal` tokens with proper scales
- Consider P3 color support (oklch is already in use)

### 2. Type scale
- Defined set of sizes with corresponding line-heights and weights
- Replace hardcoded values (`text-[27px]`, `text-[13.4px]`) with scale tokens
- Heading and body presets

### 3. Icon set
- Consolidate into `components/ui/icons.tsx`
- Standard set covering navigation, actions, status, social
- Consistent sizing, stroke width, and API (`className` prop, `currentColor`)
- Replace scattered lucide-react imports with unified icon exports where appropriate

## Design references
- Vercel/Geist: 10 scales, 10 steps each (1-3 backgrounds, 4-6 borders, 7-8 fills, 9-10 text)
- Apple HIG: semantic naming by purpose, hierarchical layering, opacity-based hierarchy
- Hybrid approach: Vercel-style numbered scales + Apple-style semantic tokens
