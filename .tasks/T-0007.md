# T-0007 -- Audit vendored auth UI -- strip unused, clean up stubs

## Context

We vendored the full `@daveyplate/better-auth-ui` component set and patched it to build against shadcn v4 (base-ui). Many components reference plugins we don't use (passkey, apiKey, captcha, anonymous sessions, multi-session) and have stubs or commented-out code. The auth-ui-provider has ~800 LOC with broad plugin support we don't need.

## Goal

Audit every vendored file. Remove components for features we're not shipping (passkey, API key management, captcha, anonymous, multi-session, one-tap). Simplify the auth-ui-provider to only support our plugins (organization, email/password, OAuth). Remove type stubs and dead code.

## Criteria

- Remove files for disabled features: passkey-button, one-tap, captcha/, email-otp forms (if not needed), recover-account
- Simplify auth-ui-provider.tsx — remove passkey/apiKey/multiSession/anonymous/oneTap branches
- Clean up stubs in hooks/auth/use-captcha.ts, lib/auth/gravatar-utils.ts
- Remove unused types from lib/auth/types/
- Remove unused localization files
- Verify `pnpm build` and `tsc --noEmit` still pass with 0 errors
- Set up prettier config for the repo
