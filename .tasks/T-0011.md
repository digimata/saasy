# T-0011 :: Fix invariant failures — sessions, slug canonicalization, membership roles

Fix the currently failing invariant tests and close the linked issue files.

## Scope

- Fix `I-0006` / `INV-SES-005` by clearing stale active workspace context after member removal
- Fix `I-0007` / `INV-WS-001` by enforcing canonical workspace slug uniqueness
- Fix `I-0008` / `INV-MEM-002` by constraining membership roles to supported values
- Update the invariant tests so the full suite passes cleanly

## Done when

- `pnpm --filter web test` passes
- `apps/web/tests/invariants/sessions.test.ts` passes
- `apps/web/tests/invariants/workspaces.test.ts` passes
- `apps/web/tests/invariants/memberships.test.ts` passes
- `I-0006`, `I-0007`, and `I-0008` can be closed

## RCA Summary

- `I-0006`: BetterAuth clears active workspace only when a user removes themselves. Admin-driven removal of another user leaves that removed user's `activeWorkspaceId` stale.
- `I-0007`: Workspace slug uniqueness is currently enforced on raw `text`, and the create/update path does not canonicalize slugs before checking uniqueness.
- `I-0008`: Membership roles are stored as unconstrained `text`, and `updateMemberRole` accepts arbitrary strings without validating against a closed role set.
