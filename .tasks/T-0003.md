# T-0003 -- Auth flows -- sign-up, sign-in, sign-out

## Context

BetterAuth is wired with the Drizzle adapter and catch-all route, but there are no auth UI pages yet. Need sign-up, sign-in, and sign-out flows using BetterAuth's client SDK.

## Goal

Build auth pages with email/password + OAuth (Google, GitHub). Wire middleware to protect dashboard routes. Create workspace on signup.

## Criteria

- Sign-up page (email/password + OAuth buttons)
- Sign-in page (email/password + OAuth buttons)
- Sign-out flow
- Auth middleware protecting `/dashboard/*` routes
- Redirect unauthenticated users to sign-in
- Local PG running with auth schema migrated
